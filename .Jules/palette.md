## 2024-04-19 - Improved Disabled States & Dynamic Interaction
**Learning:** Using purely static disabled styles is not enough; without specific `input` listeners for text areas, buttons remain enabled when fields are empty, confusing users and decreasing interface trustworthiness. Real-time feedback for dynamic interactive states (disabling buttons if input is empty) improves the overall flow. Using simple aria-labels for textareas when there are no visible distinct labels attached helps contextualize the action to screen readers.
**Action:** Always implement logic to tie interactive button disabled states to the content length/state of their corresponding input fields. Ensure that clear, explicit `aria-label` attributes or `aria-live` attributes are attached to dynamically changing elements like error messages to allow screen readers to understand UI context.
## 2024-04-26 - Dedicated Copy Buttons for Readonly Textareas
**Learning:** Users often struggle with explicitly copying from readonly textareas, especially when interacting with LLMs. Relying on users to manually select and copy text from a large box introduces friction.
**Action:** Readonly output textareas intended for external pasting should feature dedicated "Copy" buttons with explicit visual feedback ("✅ Copié !"). Their disabled state must be bound to the presence of content in the target fields.
